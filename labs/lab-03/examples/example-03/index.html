Mixins: código que se expande

1 · @include copia el bloque, no lo comparte

Compila el SCSS. Tres selectores distintos incluyen el mismo mixin con parámetros diferentes. En el CSS compilado, el bloque del mixin aparece una vez por cada @include. El mixin no crea un selector compartido: expande su contenido en el punto de uso.

2 · @content: el mixin acepta un bloque personalizado

          

            Con @content, el mixin define un "slot" donde el código
            del @include se inserta. Patrón muy usado para mixins
            de media queries: el mixin gestiona el @media, el
            @include aporta los estilos internos.
          

          

            

              
SCSS con @content

              
@mixin tablet {
  @media (min-width: 768px) {
    @content; // ← slot
  }
}

.hero {
  font-size: 1rem;

  @include tablet {
    // este bloque va al slot
    font-size: 1.5rem;
    grid-template-columns: 1fr 1fr;
  }
}

            

            

              
CSS compilado

              
.hero {
  font-size: 1rem;
}

@media (min-width: 768px) {
  .hero {
    font-size: 1.5rem;
    grid-template-columns: 1fr 1fr;
  }
}
