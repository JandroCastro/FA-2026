Nesting: estructura visual vs CSS plano

1 · Cada nivel de nesting añade un selector al resultado

Compila el SCSS. El CSS generado es plano: el compilador convierte la jerarquía visual en selectores encadenados. A más niveles de nesting, mayor especificidad — y más difícil de sobreescribir.

SCSS — estructura visual anidada
CSS compilado — selectores planos generados
Pulsa "Compilar" para ver el resultado.

Selectores generados — profundidad (partes separadas por espacio): ≤2 ok · 3 atención · ≥4 revisar

2 · El operador &: referencia al selector padre

Sin &, un selector dentro de un bloque anidado genera un selector descendente (.btn .label). Con &, se concatena directamente al padre (.btn:hover, .btn--danger).

SCSS

.btn {
  color: #fff;

  /* Sin &: genera .btn .label */
  .label {
    font-size: .875rem;
  }

  /* Con &: genera .btn:hover */
  &:hover {
    background: #4f46e5;
  }

  /* Con &: genera .btn--danger */
  &--danger {
    background: #ef4444;
  }

  /* Con &: genera .is-active.btn */
  .is-active & {
    outline: 2px solid #6366f1;
  }
}

CSS compilado

.btn {
  color: #fff;
}

/* Selector descendente */
.btn .label {
  font-size: .875rem;
}

/* Concatenado al padre */
.btn:hover {
  background: #4f46e5;
}

/* Sufijo BEM */
.btn--danger {
  background: #ef4444;
}

/* El padre queda al final */
.is-active .btn {
  outline: 2px solid #6366f1;
}
