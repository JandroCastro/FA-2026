CSS Custom Properties como API de theming

1 · Lo que no funciona vs lo que sí funciona

El Shadow DOM bloquea los selectores CSS externos. Pero las CSS Custom Properties son un caso especial: al ser valores heredables, sí atraviesan el shadow boundary. Este es el mecanismo estándar para exponer una API de theming desde dentro de un componente encapsulado.

2 · Tres temas aplicados solo desde CSS externo

El mismo componente <themed-button> renderizado tres veces. Ninguna de las diferencias visuales requiere modificar el componente: basta con definir custom properties en el CSS del documento. El componente las lee con var(--prop, fallback) desde su shadow DOM.

3 · Panel de theming en tiempo real

Cada control llama a element.style.setProperty('--prop', value) sobre el botón. El componente lo refleja al instante porque el var() dentro del shadow DOM se recalcula con el nuevo valor. Las propiedades se heredan igual que cualquier propiedad CSS heredable.
