Shadow DOM: la barrera de estilos

1 · Los estilos globales no entran — los internos no salen

Cuando un elemento tiene Shadow DOM, los selectores CSS del documento principal no pueden alcanzar los elementos que hay dentro. Y a la inversa: los estilos definidos dentro del Shadow DOM no afectan al exterior. La frontera es bidireccional. Aplica el estilo global y observa cuál de los tres botones se ve afectado.

.base-btn { background: crimson !important; color: white !important; }
1 · Light DOM

← recibe el estilo global ✅

2 · Shadow DOM · custom element

← encapsulado en <shadow-button> ✅

3 · Shadow DOM · <div> normal

← Shadow DOM no requiere custom element ✅

2 · mode: 'open' vs mode: 'closed'

Al crear un shadow root con attachShadow({ mode: 'open' }), la propiedad element.shadowRoot devuelve el shadow root desde JavaScript externo. Con mode: 'closed' devuelve null: el componente funciona, pero el exterior no puede inspeccionarlo ni modificarlo.

mode: 'open'

host.shadowRoot → ShadowRoot accesible

mode: 'closed'

host.shadowRoot → null
