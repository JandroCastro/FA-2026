customElements.define(): el primer componente

1 · Tag sin definir vs elemento registrado

El navegador acepta cualquier nombre de etiqueta desconocida: la parsea, la inserta en el DOM como HTMLElement genérico y sigue adelante. No lanza errores. Pero tampoco hace nada especial con ella. Eso cambia cuando registras una clase con customElements.define(). El CSS también lo sabe: la pseudoclase :defined distingue ambos estados.

❓ <mystery-element> — sin definir
Soy una etiqueta desconocida. El navegador me inserta en el DOM pero no sabe qué hacer conmigo.
✅ <hello-world> — registrado

2 · La regla del guión

Todo nombre de custom element debe contener al menos un guión (-). Esta norma evita colisiones con los elementos HTML nativos, presentes y futuros. El navegador lanza un DOMException si no se cumple. Prueba con boton, mi-boton y el mismo nombre dos veces.

3 · El objeto window.customElements

window.customElements es la instancia global de CustomElementRegistry. Expone métodos para consultar el registro, no solo para escribir en él. Pulsa cada tarjeta para ejecutarlo.
