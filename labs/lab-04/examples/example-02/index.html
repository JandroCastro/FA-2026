Ciclo de vida de un custom element

1 · El ciclo de vida en vivo

Un custom element no está en el DOM todo el tiempo. El constructor() se ejecuta cuando JavaScript crea el objeto en memoria. connectedCallback() se ejecuta cuando ese objeto se inserta en el árbol del documento. Son dos momentos distintos, y confundirlos es el error más frecuente: poner this.innerHTML en el constructor puede lanzar excepciones o producir resultados vacíos.

2 · observedAttributes: el guardián del callback

attributeChangedCallback() no se dispara para todos los atributos: solo para los que aparecen en el getter estático static get observedAttributes(). Sin ese getter, el callback nunca se ejecuta, aunque el atributo cambie en el DOM. Además, el callback puede dispararse antes de que el elemento entre en el DOM si el atributo se cambia mientras el elemento está en memoria.

✅ Atributos observados

Listados en observedAttributes → disparan el callback y re-renderizan el componente

❌ Atributos no observados

No listados → el atributo cambia en el DOM pero el callback no se dispara
